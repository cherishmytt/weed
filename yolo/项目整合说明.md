# YOLOv8 Pose + Spring Boot 杂草根茎定位项目整合文档

## 1. 项目目标

本项目实现了从 CropAndWeed 数据集训练到 Web 可视化推理的完整链路，目标是：

- 检测目标边界框（bbox）
- 输出每个目标的根茎关键点（stem keypoint）
- 在网页中上传图片并查看推理结果图与坐标明细

当前推荐模型路径：

- `D:/yolo/runs/cnw_crop_weed_pose/weights/best.pt`

## 3. 训练数据与标签格式

### 3.1 输入标注（CropAndWeed）

每行 CSV：

```text
Left,Top,Right,Bottom,LabelID,StemX,StemY
```

### 3.2 输出标注（YOLOv8 Pose）

每行 TXT：

```text
cls cx cy w h kx ky v
```

- `cx,cy,w,h`：归一化 bbox
- `kx,ky`：归一化关键点
- `v`：可见性（本项目用 `2`）

## 6. API 说明

### 6.1 接口

- `POST /api/predict`
- `Content-Type: multipart/form-data`

### 6.2 参数

- `file`：图片文件（必填）
- `conf`：置信度阈值（可选，默认 `0.25`）
- `classes`：类别过滤（可选，默认 `1`）

### 6.3 返回

返回 JSON 含：

- `result.detections[].bbox.{x1,y1,x2,y2}`
- `result.detections[].stem.{x,y}`
- `outputImageUrl`
- `runtimeLog`

## 7. 常见问题与排查

## 7.1 `java: 错误: 不支持发行版本 5`

已在项目中固定 Java 编译版本为 17：

- `pom.xml`：`maven-compiler-plugin` + `release/source/target=17`
- `.mvn/maven.config`：强制编译参数
- `.idea/compiler.xml`：模块字节码目标 17

若仍报错，执行：

```powershell
java -version
javac -version
mvn -version
```

并确认 Maven/IDE 运行 JDK 不是旧版本。

## 7.2 网页“检测不出结果”

优先检查以下项：

1. `classes` 过滤过严  
   先将页面 `classes` 留空测试，确认模型本身有输出。

2. `conf` 过高  
   先用 `0.1` 或 `0.05` 试跑。

3. 训练标签分布异常  
   可统计当前标签类别分布：

```powershell
$root='D:\yolo\datasets\cnw_crop_weed_pose\labels'; $counts=@{}; foreach($split in 'train','val','test'){ foreach($f in Get-ChildItem "$root\$split" -Filter *.txt){ foreach($line in Get-Content $f.FullName){ if([string]::IsNullOrWhiteSpace($line)){continue}; $cls=[int]($line.Split(' ')[0]); if($counts.ContainsKey($cls)){$counts[$cls]++}else{$counts[$cls]=1} } } }; $counts.GetEnumerator() | Sort-Object Name | ForEach-Object {"class=$($_.Name) count=$($_.Value)"}
```

如果类别明显缺失（如只剩 0/1），需要回溯数据转换脚本映射逻辑并重新生成数据后重训。
