设计文档
一、系统概述
1.1 研究背景
杂草与农作物竞争光照、水分及土壤养分，是制约作物产量与品质的重要因素之一。传统化学除草方
式虽然效率较高，但长期使用易导致土壤退化、水体污染以及农药残留等环境与食品安全问题。随着
计算机视觉与精准农业技术的发展，基于激光的物理除草方案因其非接触、无污染、可精准定位等特
性，逐渐成为该领域的研究热点。
本项目为实验室所设计的激光除草机器人构建一套完整的作业控制系统。该系统整合视觉识别与运行
状态监测功能，实现对机器人作业状态、空间位置及视觉采集数据的统一汇聚、存储与可视化展示。
1.2 系统定位与工作模式
本系统的设计范畴限定于软件层面的检测控制与数据管理，不涉及激光器、传动结构等外部硬件的电
路设计与制造。硬件交互通过预留的抽象接口层实现，具备后续硬件对接的可扩展性。
本项目所涉及的激光除草机器人采用手推车结构，由操作人员在田间手动推行。推行过程中，机载视
觉系统持续采集地面图像并执行实时杂草检测，当检测到杂草目标时自动触发激光执行定点除草。该
人机协作模式在保障作业安全性与操控可靠性的同时，降低了系统对自主导航模块的依赖。

3.3 树莓派与服务端通信机制
树莓派与 Spring Boot 服务端之间的所有通信均通过 HTTP 接口完成，无需额外的消息中间件。
上行通信（树莓派 → 服务端）：树莓派通过定时调用 HTTP POST 接口主动上报数据。
数据类型 接口 调用频率 说明
运行状态POST /api/v1/robot/status 每 5 秒 包含电量、温度、GPS、IMU 等
检测结果
与图像POST /api/v1/detection/report 每次检测
完成时multipart/form-data，含原始图
与标注图
激光执行
反馈POST
/api/v1/robot/laser/feedback执行完成
时上报指令执行结果
下行通信（服务端 → 树莓派）：树莓派通过定时轮询 HTTP GET 接口获取待执行指令。

数据类型 接口 调用频率 说明
待执行指令GET /api/v1/robot/commands/pending 每 1~2 秒 返回尚未执行的指令列表
确认执行PUT /api/v1/robot/commands/{id}/ack 执行完成时 标记指令为已执行
设计说明：下行指令采用轮询而非长连接，因激光设备当前为预留模块，对实时性要求不高，轮询方案实
现简单且对树莓派端的开发复杂度最低。

3.4.5 激光控制模块
该模块为激光硬件预留标准化控制接口。前端通过该模块下发控制指令，Spring Boot 将指令写入待
执行队列，树莓派端通过轮询 GET /api/v1/robot/commands/pending 接口获取指令并执行，执行完
成后通过 POST /api/v1/robot/laser/feedback 上报结果。当前阶段激光硬件尚未采购，系统使用
模拟驱动进行开发调试。
POST /api/v1/robot/laser/command
向激光设备下发控制指令。
请求参数（Body）：
字段 类型 必填 说明
action string 是 指令类型（见下表）
params object 否 指令参数
action 枚举值：
action 说明 params 字段
ENABLE 设备上电使能 无
DISABLE 设备断电关闭 无
FIRE 对指定坐标执行激光照射 targetX, targetY, depth, position3d, duration(ms)
STOP 立即停止当前照射 无
SET_POWER 设置激光输出功率 power (W)
AIM 瞄准指定坐标但不发射 targetX, targetY
SELF_TEST 执行设备自检 无
RESET 设备复位 无
请求示例：

{
"action":"FIRE",
"params":{
"targetX":150,
"targetY":100,
"depth":0.45,
"position3d":{"x":0.12,"y":-0.03,"z":0.45},
"duration":500,
"power":null
}
}
响应示例：
{
"code":200,
"data":{
"commandId":"cmd-20260407-001",
"status":"SENT",
"message":"指令已下发，等待设备响应 "
}
}

3.4.6 指令轮询模块
GET /api/v1/robot/commands/pending
由树莓派端定时轮询调用（建议间隔 1~2 秒），获取服务端尚未执行的指令列表。返回的指令按创建
时间升序排列。
响应示例：
{

"action":"FIRE",
"params":{

"targetY":100,
"depth":0.45,
"position3d":{"x":0.12,"y":-0.03,"z":0.45},
"duration":500
},
"createdAt":"2026-04-07 14:28:00"
}
]
}

PUT /api/v1/robot/commands/{id}/ack
由树莓派端调用，确认指定指令已执行。服务端收到确认后将该指令从待执行队列中移除。
请求参数（Body）：
字段 类型 必填 说明
result string 是 执行结果：SUCCESS / FAILED
message string 否 附加说明
响应示例：
{
"code":200,
"message":"指令已确认"
}
