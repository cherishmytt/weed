# 全球野火监测与时空分析平台系统说明文档

## 1. 文档说明

本文档用于说明当前“全球野火监测与时空分析平台”的建设背景、系统架构、功能模块、数据组织、部署方式与运行流程，便于项目交付、系统维护、功能扩展以及后续汇报使用。

本文档基于当前代码仓库与实际实现整理，覆盖前端、后端、数据库、数据导入、三维地图、分析页面、国家专题、数据大屏、用户权限与数据同步等内容。

## 2. 系统概述

### 2.1 系统名称

全球野火监测与时空分析平台

### 2.2 系统定位

本系统是一个面向全球野火火点监测、热点识别、空间态势展示与时序分析的 WebGIS 平台。系统以 NASA FIRMS 火点数据为核心数据源，结合国家边界数据、天气补充信息与多维统计分析能力，为用户提供“数据接入、地图展示、统计分析、专题研判、可视化大屏”一体化支撑能力。

### 2.3 建设目标

- 支撑全球及区域尺度的野火火点监测与展示。
- 支撑多源卫星火点数据的导入、同步、管理与查询。
- 支撑火点热点区域识别、时间回放与空间态势感知。
- 支撑综合统计分析与国家专题分析。
- 支撑面向展示与汇报场景的数据大屏。
- 支撑平台级用户管理、权限控制与安全接入。

### 2.4 适用场景

- 野火监测与研判
- 火点时空分布展示
- 区域热点识别与巡航演示
- 遥感监测成果汇报
- 教学演示与科研原型验证
- 监测指挥类可视化系统原型搭建

## 3. 系统总体架构

系统采用“前后端分离 + MySQL 数据存储 + 外部数据源接入”的技术架构。

### 3.1 架构组成

#### 前端展示层

前端基于 Vue 3 构建，负责实现登录门户、平台总览、数据管理、三维地图、数据分析、国家专题、数据大屏、用户管理等页面，并通过 Axios 调用后端接口完成数据展示与交互。

#### 后端服务层

后端基于 FastAPI 构建，负责用户认证、火点查询、数据导入、国家匹配、统计分析、热点识别、天气联动与数据同步任务管理。

#### 数据存储层

底层使用 MySQL 存储用户信息、火点数据、国家边界、导入批次、同步任务与系统配置。

#### 外部数据源

- NASA FIRMS：野火火点数据来源
- Open-Meteo：火点与热点区域的天气补充来源

### 3.2 系统逻辑链路

系统的核心数据链路如下：

1. 火点 CSV 数据通过“后台上传”或“批量脚本导入”进入系统。
2. 后端对原始数据进行时间标准化、辅助字段补充、去重与国家匹配。
3. 清洗后的火点数据写入 MySQL。
4. 前端通过查询接口、分析接口和热点接口获取数据。
5. Cesium、ECharts 与 Element Plus 负责地图展示、图表渲染与交互呈现。
6. 对重点火点或热点区域，可按需请求 Open-Meteo 天气信息。
7. 管理员可通过数据同步模块从 FIRMS 拉取最新数据并写入数据库。

- 查看同步状态

- 多区域、多数据源任务组合
- 窗口分片抓取

- 旧窗口数据清理
- 任务进度记录与状态恢复

### 5.5 数据分析模块

数据分析模块用于展示多维统计结果，是平台的日常分析主页面。

主要能力：

- 时间趋势分析
- 国家排行

- 昼夜占比
- FRP 分布
- 热点摘要

### 5.6 国家专题模块

## 6. 核心业务流程说明

### 6.1 火点数据导入流程

1. 用户上传 FIRMS CSV 文件。
2. 后端创建导入批次记录。
3. 系统标准化时间字段，生成 `acq_time_padded` 与 `acq_datetime`。
4. 系统推断并补充 `source_product`、`area_label`、`source_file`。
5. 系统按规则执行基础去重。
6. 系统使用国家边界进行空间匹配，补充国家信息。
7. 数据写入 `fire_points` 表。
8. 批次状态更新为完成。

### 6.2 国家边界导入流程

1. 上传国家边界 GeoJSON。
2. 解析行政区边界要素。
3. 提取国家名称与代码。
4. 写入 `country_boundaries`。
5. 后续火点导入与查询可使用该边界数据进行国家匹配与专题统计。

### 6.3 火点查询与展示流程

1. 前端传入区域、时间、卫星、昼夜、国家等筛选条件。
2. 后端构建统一查询参数。
3. 查询主表 `fire_points`。
4. 返回列表、范围点位、边界框点位或单条详情。
5. 前端在列表页、地图页或详情页进行展示。

### 6.4 热点识别流程

1. 系统根据时间窗读取火点数据。
2. 将火点按设定网格进行聚合。
3. 形成热点桶统计结果。
4. 输出热点列表、热点详情和大屏热点排行。

当前热点识别属于网格聚合型热点识别，兼顾性能和展示效果。

### 6.5 数据同步流程

1. 管理员选择区域和数据源。
2. 系统估算任务时间。
3. 发起同步任务。
4. 后台线程按目标组合调用 FIRMS API。
5. 拉取数据后进行去重、窗口更新和旧数据清理。
6. 写入同步任务主表与明细表。
7. 前端轮询同步状态并展示结果。

### 6.6 天气联动流程

1. 用户点击火点或热点。
2. 前端请求天气接口。
3. 后端调用 Open-Meteo。
4. 返回当前温度、降水、风速、风向等补充信息。
5. 天气结果进入短时缓存，减少重复请求。

## 7. 数据来源与数据组织

### 7.1 数据来源

当前系统主要使用以下数据：

- NASA FIRMS 火点 CSV 数据
- 国家边界 GeoJSON 数据
- Open-Meteo 实时天气数据

### 7.2 当前数据文件组织

项目支持使用 `data/firms_downloads` 目录中的 merged CSV 进行导入；公开仓库版本默认不包含大体量样例数据文件，需由使用者自行准备。

典型 merged CSV 特点：

- 已包含 `source_product`
- 已包含 `area_label`
- 已包含 `source_file`
- 已包含 `acq_time_padded`
- 已包含 `acq_datetime`

这类文件更适合直接入库和前端展示。

### 7.3 火点主表核心字段

`fire_points` 表中的重要字段包括：

- `latitude`：纬度
- `longitude`：经度
- `acq_date`：采集日期
- `acq_time`：采集时间
- `acq_time_padded`：标准化时间
- `acq_datetime`：统一时间主字段
- `satellite`：卫星名称
- `instrument`：传感器
- `confidence`：置信度
- `frp`：火辐射功率
- `daynight`：昼夜标记
- `country_name`：国家名称
- `country_code`：国家代码
- `source_product`：数据源产品
- `area_label`：区域标签
- `source_file`：来源文件
- `import_batch_id`：导入批次

### 7.4 天气字段

当前天气联动第一版使用以下字段：

- `temperature_2m`
- `precipitation`
- `wind_speed_10m`
- `wind_direction_10m`

## 14. 运维与维护建议

### 14.1 数据维护建议

- 先导入国家边界，再导入火点数据。
- 优先使用 merged CSV 作为数据导入源。
- 对全球数据建议配合 `area_label` 使用，避免不同区域数据混合统计。

### 14.2 性能建议

- 大范围查询优先通过分析聚合接口而非原始列表接口。
- 热点、大屏和国家专题建议依赖服务端缓存。
- 前端生产环境建议使用 Node.js 20.19 及以上版本。

### 14.3 扩展建议

- 可增加更多区域预设和数据源产品。
- 可扩展更细粒度的角色权限控制。
- 可增加告警规则、订阅机制和自动任务调度。
- 可增加更精细的热点算法和更丰富的专题研判模型。

## 15. 结语

当前系统已经形成一套较完整的全球野火监测与时空分析平台原型，具备数据接入、三维展示、专题分析、热点识别、天气联动、数据同步和展示大屏等核心能力。该系统既可作为教学与科研展示平台，也具备进一步扩展为业务化监测平台的良好基础。
