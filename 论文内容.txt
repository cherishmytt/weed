激光除草机器人运行监测系统
摘要
杂草与农作物竞争光照、水分及土壤养分，是制约作物产量与品质的重要因素之一。传统化学除草方式虽然效率较高，但长期使用易导致土壤退化、水体污染以及农药残留等环境与食品安全问题。随着计算机视觉与精准农业技术的发展，基于激光的物理除草方案因其非接触、无污染、可精准定位等特性，逐渐成为该领域的研究热点。然而，激光除草机器人在实际作业过程中，如何对机器人的运行状态、检测结果、激光执行等关键信息进行有效监控与管理，是保障系统稳定运行的重要问题。
本课题针对激光除草机器人作业过程中的监控管理需求，设计并实现了一套基于Web的激光除草机器人运行监测系统。系统采用B/S架构，通过软件模拟的方式实现机载端与服务端的通信交互，完成机器人运行状态上报、杂草检测结果展示、激光控制指令下发等核心功能。前端采用Vue 3与Element Plus组件库实现数据可视化界面，后端采用Spring Boot框架提供RESTful API服务，通过WebSocket实现实时数据推送。本系统的创新点在于采用软件仿真的方式，在不依赖实际硬件设备的情况下，完整验证了激光除草机器人监控系统的业务流程与技术可行性，为后续硬件系统的集成与部署提供了软件基础。
本课题所设计并实现的激光除草机器人运行监测系统，实现了机器人状态实时监控、检测记录管理、激光控制指令管理等核心功能。系统界面简洁直观，操作便捷，能够有效帮助操作人员远程监控机器人的作业状态，提升激光除草机器人的作业管理效率。
关键词：运行监测系统，除草机器人，Web系统

Laser Weeding Robot Operation Monitoring System
Abstract
Weeds compete with crops for sunlight, water, and soil nutrients, which is one of the important factors restricting crop yield and quality. Although traditional chemical weeding methods are efficient, long-term use can easily lead to environmental and food safety problems such as soil degradation, water pollution, and pesticide residues. With the development of computer vision and precision agriculture technology, laser-based physical weeding solutions have gradually become a research hotspot in this field due to their non-contact, pollution-free, and precise positioning characteristics. However, during the actual operation of laser weeding robots, how to effectively monitor and manage key information such as robot operation status, detection results, and laser execution is an important issue to ensure stable system operation.
This thesis designs and implements a Web-based laser weeding robot operation monitoring system to address the monitoring and management requirements during the operation of laser weeding robots. The system adopts a B/S architecture, realizes communication interaction between the on-board end and the server through software simulation, and completes core functions such as robot operation status reporting, weed detection result display, and laser control command distribution. The front-end uses Vue 3 and Element Plus component library to implement a data visualization interface, and the back-end uses Spring Boot framework to provide RESTful API services, realizing real-time data push through WebSocket. The innovation of this system is that it adopts software simulation to fully verify the business process and technical feasibility of the laser weeding robot monitoring system without relying on actual hardware equipment, providing a software foundation for subsequent hardware system integration and deployment.
The laser weeding robot operation monitoring system designed and implemented in this thesis realizes core functions such as real-time robot status monitoring, detection record management, and laser control command management. The system interface is simple, intuitive, and easy to operate, which can effectively help operators remotely monitor the robot's operation status and improve the operation management efficiency of laser weeding robots.

Key Words: Operation monitoring system, Weeding robot, Web system



第1章 绪论
1.1 研究背景与意义
农业是国民经济的基础，农作物的产量与品质直接关系到粮食安全与人民生活水平。在农业生产过程中，杂草防治是田间管理的重要环节。据统计，全球每年因杂草造成的农作物产量损失约占总产量的34%[1]。传统的杂草防治方式主要包括人工除草、机械除草和化学除草。人工除草效率低、劳动强度大；机械除草容易造成土壤结构破坏；化学除草剂的大量使用会造成土壤污染、水体富营养化、农药残留等生态环境问题[2]。
随着计算机视觉、人工智能与机器人技术的快速发展，精准农业技术为杂草防治提供了新的解决方案。基于机器视觉的杂草检测与物理除草技术，可以实现杂草的精准识别与定点清除[3]。其中，激光除草技术因其非接触、无污染、作用时间短、可精准控制等优点，成为近年来精准农业领域的研究热点[4]。
激光除草机器人是一种集成了机器视觉、运动控制、激光技术等多学科技术的智能农业装备。机器人在实际作业过程中需要对运行状态、检测结果、激光执行情况等进行实时监控与管理。传统的监控方式依赖于本地控制台显示，无法实现远程监控与集中管理[5]。
Web技术的发展为远程监控系统提供了成熟的技术方案。基于B/S架构的Web监控系统，用户只需通过浏览器即可访问系统，无需安装专用客户端，具有跨平台、易部署、维护方便等优点[6]。WebSocket等实时通信技术的应用，使得Web系统能够实现数据的实时推送，满足监控系统的实时性需求。
本课题针对激光除草机器人的监控管理需求，设计并实现一套基于Web的激光除草机器人运行监测系统。系统采用软件仿真的方式，在不依赖实际硬件设备的情况下，验证激光除草机器人监控系统的业务流程与技术可行性。
1.2 国内外研究现状
激光除草技术自20世纪80年代开始研究，进入21世纪后，随着计算机视觉与机器人技术的发展，逐渐向自动化、智能化方向发展[7]。国外方面，美国Carbon Robotics公司开发的Autonomous LaserWeeder机器人已实现商业化应用，每小时可处理20万株杂草[8]。国内方面，中国农业大学、浙江大学等高校开展了激光除草相关研究，但大多处于实验室阶段，距离产业化应用还有一定差距[10][11]。
在农业机器人监控系统方面，美国John Deere公司、国内中国农业机械化科学研究院等都开发了相应的远程监控系统[12][13]。Web监控系统技术已相当成熟，Vue、Spring Boot、WebSocket等技术的应用，使得实时监控成为可能[15][17][18]。

综上所述，激光除草技术与Web监控系统技术都已取得长足发展，但专门针对激光除草机器人的Web运行监测系统研究相对较少。本课题将Web技术与激光除草机器人监控需求相结合，具有重要的研究价值。

第2章 相关技术介绍
2.1 前端开发技术
Vue 3是一套用于构建用户界面的渐进式JavaScript框架，由尤雨溪于2014年首次发布，2020年9月发布Vue 3版本[22]。Vue 3采用Composition API作为新的代码组织方式，具有更好的逻辑复用能力与类型推导支持。其使用Proxy替代Object.defineProperty实现响应式系统，提升了性能并解决了Vue 2的局限性。Vue 3还对虚拟DOM进行了重写，引入了PatchFlag、Block Tree等优化技术，大幅提升渲染性能。
Vite是新一代前端构建工具，利用浏览器原生支持的ES Module特性，在开发环境下无需打包，实现秒级启动[23]。Vite在开发模式下采用按需编译，只编译当前页面需要的模块，大大提升热更新速度。
ECharts是由百度开发的开源可视化库，2018年捐赠给Apache基金会，提供丰富的图表类型，包括折线图、柱状图、散点图、地图等，支持交互操作[24]。
Element Plus是基于Vue 3的UI组件库，提供丰富的UI组件，设计遵循一致规范，具有统一的视觉风格，支持TypeScript类型[25]。
在本系统中，前端采用Vue 3作为核心框架，Vite作为构建工具，ECharts实现数据可视化，Element Plus作为UI组件库。

2.2 后端开发技术
Spring Boot是基于Spring框架的快速开发框架，采用"约定优于配置"的理念，提供开箱即用的配置，内嵌Servlet容器，无需部署WAR文件[26]。Spring Boot提供丰富的Starter依赖和生产就绪特性，便于应用运维管理。
MyBatis Plus是在MyBatis基础上增强的开源框架，提供通用的Mapper与Service，无需编写SQL语句即可实现单表CRUD操作，支持分页插件、性能分析插件等[27]。
Sa-Token是轻量级Java权限认证框架，解决登录认证、权限认证、Session会话等问题，API简洁易用，支持多种Token生成策略和Redis等外部存储[28]。
本系统后端采用Spring Boot作为核心框架，MyBatis Plus作为数据持久层框架，Sa-Token实现权限认证。

2.3 数据库技术
MySQL是流行的关系型数据库管理系统，具有开源、免费、性能高、可靠性好等优点，支持标准SQL语言和多种存储引擎[29]。InnoDB存储引擎支持事务、外键、行级锁等特性，适合大多数业务场景。MySQL支持主从复制、读写分离等高可用架构，可满足高并发需求。
本系统采用MySQL存储用户信息、机器人状态、检测记录、激光指令等业务数据。
2.4 通信协议技术
HTTP是Web应用中基础的协议，基于TCP/IP，采用请求-响应模型，支持GET、POST、PUT、DELETE等方法[30]。RESTful API基于HTTP协议设计，使用HTTP方法表示操作类型，具有简洁、易读、易扩展等优点。
WebSocket是HTML5提供的全双工通信协议，解决了HTTP协议服务器无法主动推送的问题[31]。WebSocket握手采用HTTP协议，握手完成后建立持久TCP连接，双方可随时发送数据，开销小、实时性高，适合实时通信场景。
本系统机载仿真端与后端服务采用HTTP协议通信，后端与前端实时数据推送采用WebSocket协议。

2.5 目标检测技术
YOLO是基于深度学习的目标检测算法，由Joseph Redmon等人于2015年首次提出，将目标检测转化为回归问题，实现端到端检测[32]。YOLO系列算法检测速度快，已发布多个版本，在农业领域广泛应用于杂草检测、病虫害识别等场景。
YOLOv8是Ultralytics公司2023年发布的最新版本，提供目标检测、实例分割、姿态估计等功能[33]。姿态估计模型可检测目标边界框与关键点坐标，为激光除草提供精准定位信息。
本系统集成YOLOv8姿态估计模型用于杂草检测，获取杂草边界框与茎干关键点信息。

第3章 系统需求分析
3.1 系统概述
激光除草机器人运行监测系统是一套针对激光除草机器人作业过程中的监控管理需求而设计的Web系统。系统旨在实现机器人运行状态的实时监控、杂草检测结果的管理与展示、激光控制指令的下发与执行跟踪等功能，帮助操作人员远程管理激光除草机器人的作业过程。
本系统采用软件仿真的方式，在不依赖实际硬件设备的情况下，验证激光除草机器人监控系统的业务流程与技术可行性。系统通过Python脚本模拟树莓派机载端的行为，实现状态上报、检测上报、指令轮询等功能，与后端服务进行通信交互。
系统采用三层架构设计，包括机载仿真层、后端服务层与前端展示层。机载仿真层负责模拟机器人的运行，上报状态数据与检测结果，轮询并执行指令。后端服务层负责数据的存储与处理，提供API服务与实时推送。前端展示层负责将数据以可视化的形式展示给用户，提供用户交互界面。
3.2 功能需求分析
本系统是基于B/S架构的激光除草机器人运行监测系统，主要包含用户认证、机器人状态监控、检测记录管理、激光控制、仪表盘统计、状态历史查询、轨迹地图展示、YOLO在线检测、操作日志管理等功能模块。用户认证模块负责登录登出与权限验证；机器人状态监控模块通过WebSocket实时推送机器人电量、温度、GPS坐标等运行数据；检测记录管理模块存储和展示杂草检测结果；激光控制模块实现激光指令下发与执行跟踪；仪表盘模块以图表形式展示系统统计数据；状态历史模块支持历史数据查询与导出；轨迹地图模块展示机器人运动轨迹并支持回放；YOLO在线检测模块提供图片上传与杂草检测功能；操作日志模块记录激光控制操作历史。系统用例图如图3-1所示。

3.3 非功能需求分析
3.3.1 性能需求
性能是衡量系统质量的重要指标，本系统的性能需求主要包括以下几个方面：
（1）响应时间：系统接口的平均响应时间应小于500ms，95%的请求响应时间应小于1s。前端页面的加载时间应小于2s，确保用户操作的流畅性。
（2）并发能力：系统应支持至少50个并发用户同时在线访问，在并发访问情况下系统仍能保持稳定运行。
（3）数据上报频率：机载仿真端状态上报频率为每5秒一次，系统应能稳定处理该频率的数据上报，不出现数据丢失或处理延迟。
（4）实时推送延迟：WebSocket实时推送的延迟应小于100ms，确保前端展示的实时性。
3.3.2 安全性需求
安全性是系统设计的重要考虑因素，本系统的安全性需求主要包括以下几个方面：
（1）身份认证：系统所有受保护的接口都需要进行身份认证，防止未授权访问。
（2）密码安全：用户密码应采用加密存储，不允许明文存储密码。推荐使用BCrypt等安全的哈希算法进行密码加密。
（3）Token安全：认证Token应具有有效期，过期后需要重新登录。Token应采用安全的生成算法，防止被伪造。
（4）数据传输安全：系统应支持HTTPS协议，保证数据在传输过程中的安全性。
（5）接口防刷：系统应具备接口防刷机制，防止恶意调用接口造成系统压力。
3.3.3 可用性需求
可用性是指系统正常运行的时间比例，本系统的可用性需求主要包括以下几个方面：
（1）系统可用性：系统的可用性应达到99%以上，即全年停机时间不超过87.6小时。
（2）界面友好性：系统界面应简洁直观，操作便捷，符合用户的使用习惯。用户应能在短时间内掌握系统的基本操作。
（3）容错能力：系统应具备良好的容错能力，对于用户的非法输入或异常操作，系统应能给出友好的提示，而不是崩溃或抛出异常信息。
（4）兼容性：系统应兼容主流的浏览器，包括Chrome、Firefox、Safari、Edge等，确保在不同浏览器下都能正常显示与操作。
3.3.4 可扩展性需求
可扩展性是指系统应对需求变化的能力，本系统的可扩展性需求主要包括以下几个方面：
（1）功能扩展：系统应采用模块化设计，各模块之间耦合度低，便于后续功能的扩展与升级。
（2）硬件扩展：系统设计应考虑后续实际硬件设备的接入，当有真实树莓派与激光设备时，应能方便地接入系统，不需要对系统架构进行大的改动。
（3）数据扩展：随着系统运行时间的增长，数据量会不断增加，系统应能应对数据量增长带来的挑战，支持数据的高效存储与查询。
（4）并发扩展：系统应支持水平扩展，当并发用户数增加时，可以通过增加服务器节点的方式提升系统的处理能力。
3.4 本章小结
本章对激光除草机器人运行监测系统进行了全面的需求分析。首先对系统进行了概述，明确了系统的定位与目标。然后详细分析了系统的功能需求，包括用户认证模块、机器人状态监控模块、检测记录管理模块、激光控制模块、仪表盘统计模块等五个核心模块的具体功能需求。最后分析了系统的非功能需求，包括性能需求、安全性需求、可用性需求与可扩展性需求。需求分析是系统设计与实现的基础，本章的内容为后续的系统设计提供了明确的依据。

第4章 系统设计
4.1 系统总体架构设计
4.1.1 三层架构设计
本系统采用经典的三层B/S架构设计，从上到下依次为前端展示层、后端服务层与机载仿真层。三层架构具有职责清晰、耦合度低、可扩展性强等优点，便于系统的开发、维护与升级。
系统总体架构图如下所示：
各层的职责如下：
（1）前端展示层：负责与用户进行交互，展示系统数据，接收用户操作。前端采用Vue 3框架开发，使用Element Plus作为UI组件库，使用ECharts进行数据可视化。前端与后端通过HTTP协议进行API调用，通过WebSocket协议接收实时数据推送。
（2）后端服务层：负责业务逻辑处理、数据存储与API服务提供。后端采用Spring Boot框架开发，提供RESTful API接口与WebSocket服务。后端包含用户认证、状态管理、检测记录、激光控制等核心业务模块，使用MySQL数据库进行数据持久化存储。
（3）机载仿真层：负责模拟树莓派机载端的行为，与后端服务进行通信。机载仿真端采用Python开发，使用多线程技术实现状态上报、检测上报、指令轮询等并发任务。使用Rich库实现终端实时可视化面板，展示仿真端的运行状态。
4.1.2 系统工作流程设计
系统的核心工作流程描述了激光除草机器人从登录开始到完成作业的完整过程。系统工作流程图如下所示：
系统的核心工作流程如下：
（1）用户通过前端页面输入用户名与密码进行登录，前端将登录请求发送至后端，后端验证凭据正确性后返回认证Token。
（2）前端建立与后端的WebSocket连接，用于接收实时数据推送。
（3）机载仿真端登录后端获取认证Token，建立通信会话。
（4）机载仿真端每5秒上报一次机器人运行状态，后端保存状态数据并通过WebSocket推送给前端。
（5）机载仿真端完成杂草检测后，上报检测结果与图像，后端保存检测记录并通过WebSocket推送给前端。
（6）用户通过前端界面下发激光控制指令，后端保存指令并返回成功。
（7）机载仿真端每1~2秒轮询一次待执行指令，获取指令并执行。
（8）机载仿真端执行完指令后，向服务端发送执行反馈，后端更新指令状态并通过WebSocket推送给前端。

4.5 机载仿真端设计
机载仿真端是系统的重要组成部分，负责模拟真实机器人的运行行为，包括状态上报、杂草检测、指令执行等功能。通过软件仿真的方式，可以在不依赖真实硬件设备的情况下，完整验证系统的功能与性能。
4.5.1 多线程架构设计
机载仿真端采用多线程架构设计，使用三个独立的线程分别处理状态上报、检测上报与指令轮询任务，模拟真实系统中的并行操作。机载仿真端的线程架构如图4-9所示。
机载仿真端的线程架构：
（1）主线程：负责初始化仿真器，加载配置参数，建立与后端服务的HTTP会话，启动其他工作线程，管理程序生命周期，处理退出信号与资源清理。
（2）状态上报线程：负责定时采集机器人状态数据并上报至服务端，每5秒执行一次。状态数据包括电量、温度、CPU使用率、GPS坐标、速度、姿态角等，每次上报时加入随机变化，模拟真实机器人的状态变化。
（3）检测上报线程：负责模拟杂草检测过程，从样本库中随机选择测试图片，调用YOLO推理服务进行杂草检测，解析检测结果并生成标注图片，上报检测结果至服务端。检测间隔在5秒到10秒之间随机，模拟真实作业中的不定期检测。
（4）指令轮询线程：负责定时轮询待执行的激光指令，获取并执行指令，轮询间隔在1秒到2秒之间随机。获取到指令后模拟执行过程，执行完成后上报执行结果并发送ACK确认。
各线程之间通过线程安全的队列进行数据通信，避免竞态条件。使用事件标志实现线程间的同步与停止控制。
4.5.2 通信机制设计
机载仿真端与后端服务之间的通信采用HTTP协议。通信机制设计如图4-17所示。

由于激光控制模块为预留模块，对实时性要求不高，采用轮询方式获取指令可以简化实现，降低开发复杂度。
上行通信（机载仿真端 → 后端服务）：机载仿真端通过调用HTTP POST接口主动上报数据。
上行通信的数据流：
（1）运行状态上报：调用`POST /api/v1/robot/status`接口，每5秒上报一次，上报内容包括电量、温度、CPU使用率、GPS坐标、速度、姿态角等。
（2）检测结果上报：调用`POST /api/v1/detection/report`接口，在每次检测完成时上报，上报内容包括原始图片、标注图片、杂草数量、每个检测结果的详细信息。
（3）激光执行反馈上报：调用`POST /api/v1/robot/laser/feedback`接口，在执行完激光指令后上报，上报内容包括指令ID、执行结果、执行时长、错误信息等。
下行通信（后端服务 → 机载仿真端）：机载仿真端通过定时调用HTTP GET接口轮询待执行指令。
下行通信的数据流：
（1）待执行指令获取：调用`GET /api/v1/robot/commands/pending`接口，每1~2秒轮询一次，获取待执行的激光指令列表。
（2）指令执行确认：调用`PUT /api/v1/robot/commands/{id}/ack`接口，在执行完指令后调用，标记指令为已执行。

5.1 开发环境搭建

后端开发使用Java 17语言，开发工具为IntelliJ IDEA，构建工具为Maven。后端主要依赖包括Spring Boot Web框架、Spring Security安全认证框架、WebSocket实时通信支持、MyBatis Plus数据持久化增强工具、MySQL数据库驱动、JWT令牌生成与验证工具、OpenAPI文档生成工具以及Lombok简化Java代码工具。
后端配置文件包含服务器端口配置、数据库连接配置、文件上传大小限制配置、MyBatis Plus配置、JWT密钥和有效期配置、文件存储路径配置以及YOLO模型相关配置（Python可执行文件路径、推理脚本路径、模型权重路径、输出目录等）。
5.1.2 前端开发环境
前端开发使用Node.js 18+环境，开发工具为Visual Studio Code，构建工具为Vite。前端主要依赖包括Vue 3核心框架、Vue Router路由管理、Pinia状态管理、Element Plus UI组件库、ECharts图表可视化以及Axios HTTP请求库。
前端项目结构清晰，包含API接口目录、静态资源目录、通用组件目录、布局组件目录、路由配置目录、状态管理目录、工具函数目录、页面组件目录以及根组件和入口文件。
5.2 模块实现
5.2.1 用户认证与授权模块
用户认证与授权模块负责系统的安全认证和权限控制，采用Spring Security + JWT的方式实现。用户密码使用BCrypt算法加密存储，确保用户信息安全。
登录界面包含用户名输入框、密码输入框和登录按钮。用户输入正确的用户名和密码后点击登录按钮，系统验证成功后生成JWT令牌并返回给前端，令牌有效期为24小时。前端将令牌保存到本地存储中，并自动跳转到首页。
前端在后续请求的Authorization头中携带该令牌，后端通过JWT认证过滤器进行验证。若令牌过期或无效，系统会返回401未授权状态码，前端自动跳转到登录页面。
登出功能实现了令牌黑名单管理，用户点击登出按钮后，系统将令牌加入黑名单，防止已过期令牌被重复使用，同时前端清除本地存储的令牌并跳转到登录页面。
 
5.2.2 机器人状态监测模块
机器人状态监测模块负责机器人状态数据的接收、存储、查询和实时展示，是系统的核心功能之一。
状态上报功能实现了机器人状态数据的接收和存储，树莓派每5秒上报一次状态数据，包括电量、速度、温度、激光状态、CPU使用率、GPS坐标、IMU数据等。状态数据上报时会更新机器人当前状态和激光连接状态，并通过WebSocket实时广播状态更新信息。
仪表盘页面是状态监测的核心界面，采用玻璃态设计风格，包含以下主要部分：
（1）核心实时状态卡片：以网格布局展示机器人状态、速度、坐标、激光状态、电量、CPU使用率、温度等实时数据。机器人状态分为离线、待机、作业中、故障等，用不同颜色标识。实时状态卡片右上角有"实时"标识，显示数据正在实时更新。
（2）设备运行参数图表：使用ECharts绘制电量、CPU使用率、温度的实时趋势图，图表数据随状态上报自动更新，最多显示最近20条数据。用户可以通过图表直观地了解设备运行状况的变化趋势。
（3）今日检测目标分布：使用饼图展示作物和杂草的数量分布，帮助用户快速了解当天的检测情况。
（4）今日作业统计：展示总检测次数、杂草清除总量、作物识别总量、激光作业次数等统计数据，数据随检测结果上报实时更新。
（5）IMU姿态可视化：展示机器人的IMU姿态数据，包括加速度计、陀螺仪、磁力计等信息。

状态历史页面提供机器人状态历史的查询、筛选和导出功能：
（1）时间范围筛选器：支持快捷选择（今天、昨天、近7天、近30天）和自定义时间范围，用户可以灵活选择需要查询的时间段。
（2）高级筛选面板：提供激光状态、电量、温度、CPU使用率等条件筛选。用户可以设置电量最大值、温度最小值、CPU使用率最小值等异常条件，快速筛选出异常状态记录。
（3）数据表格：以表格形式展示状态历史记录，包括时间、电量、速度、温度、激光状态、CPU使用率、经纬度等信息。表格支持分页显示，每页显示20条记录。
（4）排序功能：用户可以按时间、电量、温度、CPU使用率等字段进行升序或降序排序。
（5）统计信息：展示总记录数、最高温度、平均温度、最低电量、平均CPU、最高CPU、平均速度、激光开启次数等统计数据，帮助用户快速了解系统运行概况。
（6）导出功能：支持导出选中记录或筛选结果为CSV文件，方便用户进行后续分析。
（7）快捷查看按钮：提供快速查看低电量（&lt;20%）、高温（&gt;40℃）、高CPU（&gt;80%）记录的快捷按钮，方便用户快速定位异常数据。

5.2.3 轨迹管理模块
轨迹管理模块负责机器人运动轨迹的记录、查询和可视化展示。
轨迹地图页面使用高德地图API实现机器人运动轨迹的可视化展示：
（1）时间范围选择：用户可以选择需要查询的轨迹时间段，支持快捷选择和自定义时间范围。
（2）地图切换：支持标准地图、卫星地图、卫星+路网地图三种模式，用户可以根据需要切换不同的地图视图。
（3）轨迹显示：在地图上绘制机器人的运动轨迹，轨迹线采用渐变色，从绿色到红色表示时间从早到晚。
（4）轨迹点信息：点击轨迹点可以查看详细信息，包括时间、电量、速度、温度、激光状态、CPU使用率等。
（5）轨迹统计：显示总距离、平均速度、最大速度、作业时长、起始位置、结束位置等信息。
（6）轨迹回放：支持播放/暂停、速度调节（0.5x、1x、2x、5x）、进度拖动的轨迹回放功能。回放时，地图上会动态显示机器人的位置变化，并更新当前点的信息。

5.2.4 激光设备控制模块
激光设备控制模块负责激光设备的指令下发、执行反馈和日志记录，采用轮询机制实现指令下发。

激光控制页面提供激光设备的控制功能：
（1）设备状态卡片：显示连接状态、当前状态、功率、上次操作时间等信息。连接状态分为已连接和未连接，当前状态分为待机、作业中、故障等。
（2）指令控制面板：提供各种控制按钮，包括开启激光、关闭激光、开始照射、停止照射、设备自检、设备复位、设置功率等。点击按钮后，系统会下发相应的指令，并显示指令执行状态。
（3）坐标与参数设置区域：支持自定义目标坐标（X、Y、Z）、照射时长、功率设置等参数，用户可以根据需要设置精细的控制参数。
（4）执行日志区域：显示最近的指令执行记录，包括指令ID、动作、结果、消息、时间等信息，方便用户查看指令执行历史。
指令下发功能实现了控制指令的创建和存储，为每条指令生成唯一的指令ID，记录指令动作、参数、状态和创建时间。指令状态包括待执行、执行中、已完成、失败等。
树莓派每1-2秒查询一次待执行指令，获取指令后模拟执行过程，根据指令类型设置不同的执行时长。执行完成后，上报执行反馈，系统更新指令状态并通过WebSocket广播执行反馈信息。
激光日志页面提供激光设备操作日志的查询和筛选功能，用户可以按时间范围、操作类型、执行结果等条件筛选日志，查询结果支持分页显示。

5.2.5 杂草检测模块

杂草检测模块负责检测结果的上报、存储、查询以及在线检测功能，整合了YOLO模型推理服务。

杂草检测页面提供在线图片检测功能：

（1）图片上传区域：支持拖拽上传和点击选择，用户可以上传本地图片进行检测。

（2）参数设置区域：提供置信度滑块（0.01-1.0，默认0.25）和类别选择（全部、作物、杂草），用户可以根据需要调整检测参数。

（3）结果展示区域：并排显示原图和标注图，支持缩放查看。标注图上用不同颜色的边界框和关键点标识检测到的作物和杂草，边界框旁边标注类别名称和置信度。

（4）检测统计区域：显示总目标数、杂草数量、作物数量、推理耗时等信息。

（5）目标列表区域：以表格形式展示每个检测目标的详细信息，包括类别、置信度、像素坐标、关键点信息、深度、三维坐标等。表格支持按类别筛选，方便用户查看特定类别的检测目标。

YOLO在线检测功能实现了图片上传、模型推理和结果解析。用户上传图片后，系统生成唯一的请求ID，保存图片到指定目录，然后调用Python推理脚本执行模型推理。推理完成后，系统解析JSON格式的推理结果，生成标注图片，并返回推理结果、标注图片、原始图片和运行日志给前端。

树莓派每5-10秒随机选择测试图片进行检测，调用后端YOLO检测接口进行推理，获取检测结果后，生成标注图片并上报给后端系统。上报的检测结果包含图片信息、检测目标列表、统计信息等。

检测记录页面提供杂草检测记录的查询和筛选功能，用户可以按时间范围、检测类别等条件筛选记录，查询结果支持分页显示。点击记录可以查看检测详情，包括标注图片、检测统计、目标列表等信息。


5.2.6 WebSocket实时推送模块

WebSocket实时推送模块负责将机器人状态更新、检测结果、指令执行反馈等信息实时推送给前端。

WebSocket处理器管理所有客户端连接，当有新的状态更新、检测结果或指令执行反馈时，通过WebSocket广播给所有连接的客户端。广播的消息类型包括状态更新、检测结果、指令执行反馈等，前端根据消息类型更新相应的界面。

前端WebSocket通信工具封装了WebSocket连接管理、消息监听和重连逻辑。当WebSocket连接断开时，系统会自动尝试重连，最多重连10次，每次重连间隔3秒。前端通过监听不同类型的消息，实时更新仪表盘、状态历史、激光控制等页面的数据。

5.2.7 机载仿真端模块

机载仿真端负责模拟真实机器人的运行行为，包括状态上报、杂草检测、指令执行等功能，采用多线程架构，使用Python语言实现。

机载仿真端的多线程架构包括主线程、状态上报线程、检测上报线程、指令轮询线程：

（1）主线程：负责初始化仿真器、加载配置、管理程序生命周期。

（2）状态上报线程：每5秒上报一次机器人状态数据，模拟机器人状态数据的采集和上报，包括电量、温度、CPU使用率、GPS坐标、速度、IMU数据等。每次上报时在合理范围内随机变化这些参数，模拟真实运行状态。电量缓慢下降，温度和CPU使用率在一定范围内波动，GPS坐标缓慢移动模拟机器人前进，IMU数据包含加速度计、陀螺仪、磁力计信息。

（3）检测上报线程：每5-10秒从测试图片库中随机选择图片，调用后端YOLO检测接口进行推理，获取检测结果后，生成标注图片并上报给后端系统。上报的检测结果包含图片信息、检测目标列表、统计信息等。

（4）指令轮询线程：每1-2秒查询待执行指令，获取指令后模拟执行过程，根据指令类型设置不同的执行时长。执行完成后，上报执行反馈并确认指令。指令执行结果包括成功、失败、超时等，执行消息包含详细的执行信息。

5.3.1 测试环境

系统测试环境配置包括Windows 10或macOS 12+操作系统，Chrome 120+、Edge 120+或Firefox 115+浏览器，OpenJDK 17，Node.js 18.x LTS，MySQL 8.0+，Python 3.10+。

5.3.2 功能测试

功能测试对系统的各项功能进行验证，主要测试用例包括：

（1）用户认证模块：测试用户登录、错误密码登录、Token过期、用户登出等功能。测试结果表明，用户认证模块功能正常，安全性良好。

（2）机器人状态监测模块：测试实时状态显示、图表更新、统计数据、状态历史查询、筛选、排序、导出等功能。测试结果表明，仪表盘能够实时更新状态数据，图表数据随状态上报正确更新，统计数据准确显示，状态历史模块的筛选功能完善，排序正确，导出功能正常。

（3）轨迹管理模块：测试轨迹显示、轨迹回放等功能。测试结果表明，轨迹地图能够正确绘制机器人运动轨迹，轨迹回放功能正常。

（4）激光设备控制模块：测试指令下发、状态查询、日志查询等功能。测试结果表明，激光控制模块能够正确下发达指令，查询设备状态，记录操作日志。

（5）杂草检测模块：测试图片上传、模型推理、参数调整、检测记录查询等功能。测试结果表明，杂草检测模块能够正确上传图片，调用模型推理，参数调整后检测结果随之变化，检测记录查询功能正常。

（6）WebSocket模块：测试实时推送功能。测试结果表明，WebSocket能够实时推送状态更新、检测结果和指令执行反馈。

（7）机载仿真端模块：测试状态上报、检测上报、指令执行等功能。测试结果表明，机载仿真端能够正常模拟机器人运行行为，上报状态数据和检测结果，执行指令并上报反馈。

5.3.3 性能测试

性能测试主要测试系统的响应时间和并发处理能力。使用JMeter工具进行测试，测试结果表明：

（1）用户登录接口在50并发用户下，平均响应时间为120ms，95%响应时间为180ms，成功率为100%。

（2）查询最新状态接口在100并发用户下，平均响应时间为45ms，95%响应时间为80ms，成功率为100%。

（3）查询状态历史接口（1000条记录）在50并发用户下，平均响应时间为150ms，95%响应时间为250ms，成功率为100%。

（4）上报机器人状态接口在100并发用户下，平均响应时间为60ms，95%响应时间为120ms，成功率为100%。

（5）上报检测结果接口在30并发用户下，平均响应时间为350ms，95%响应时间为500ms，成功率为100%。

（6）YOLO在线检测接口在10并发用户下，平均响应时间为1200ms，95%响应时间为1800ms，成功率为100%。

测试结果表明，系统在100并发用户下的响应时间均在可接受范围内，各项接口的成功率达到100%，系统具有良好的性能表现。

5.4 本章小结

本章详细介绍了激光除草机器人运行监测系统的实现与测试。首先介绍了开发环境的搭建，包括后端和前端的技术栈和依赖配置。然后详细阐述了各功能模块的实现，包括用户认证与授权、机器人状态监测、轨迹管理、激光设备控制、杂草检测、WebSocket实时推送、机载仿真端等模块的功能实现和界面设计。最后进行了系统测试，包括功能测试、性能测试和兼容性测试，测试结果表明系统功能完善、性能良好、兼容性强，达到了预期的设计目标。
